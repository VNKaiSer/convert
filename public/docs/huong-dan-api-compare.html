Hướng dẫn sử dụng API Compare

Tab API Compare trong công cụ JSON Compare (/json-compare)

Mục đích

API Compare giúp bạn:

  1. Gọi 2 API (Endpoint 1 và Endpoint 2) với cùng cấu hình request
  2. Lấy response, tự động gộp kết quả (khi chạy batch)
  3. So sánh (diff) JSON giữa hai phía và xem field nào khác

Phù hợp khi regression test giữa 2 môi trường, 2 version API, hoặc so sánh output từ nhiều bản ghi input (vd. hàng trăm / hàng nghìn pcbRaw).

Quy trình chuẩn (ví dụ pcbRaws + icores)

  1. Upload TXT (JSONL) → Convert → converted/*.json
  2. Load vào biến pcbRaws, icores
  3. Cấu hình Endpoint 1 & 2 — body dùng {{pcbRaw}}, {{icoreRaw}}
  4. Chọn Array 1 + Body field 1 (và Array 2 nếu cần)
  5. Call Compare with Variable
  6. Gộp response → JSON 1 / JSON 2 → Auto compare → Differences

Bước 1 — Chuẩn bị dữ liệu input

  1. Vào tab API Compare
  2. Ở Variables, thêm biến array (vd. pcbRaws, icores)
  3. Ở TXT → JSON Converter: Upload TXT (mỗi dòng = 1 JSON object) → file lưu vào converted/ (vd. pcb.json, icore.json)
  4. Với mỗi file convert: chọn Converted file → Variable key tương ứng → Load to variable

Mỗi biến có dạng JSON array: [{...}, {...}, ...].

Gợi ý đặt tên: array số nhiều (pcbRaws, icores) → field body số ít (pcbRaw, icoreRaw). Tool tự gợi ý body field khi tên array kết thúc bằng s.

Bước 2 — Cấu hình Endpoint

TrườngMô tả
MethodGET, POST, PUT, PATCH, DELETE
URLURL đầy đủ, hỗ trợ {{tên_biến}}
Headers (JSON)Object JSON, hỗ trợ biến
Body (JSON)Chỉ hiện với POST / PUT / PATCH

Body gợi ý (1 field):

{
  "pcbRaw": "{{pcbRaw}}"
}

Body gợi ý (2 field — dùng cùng lúc pcbRaws + icores):

{
  "data": {
    "pcbRaw": "{{pcbRaw}}",
    "internalFacilitiesRaw": "{{icoreRaw}}",
    "declaredIncome": 20000000,
    "productType": "REVI_CREDIT",
    "productCode": "REVI_CREDIT_OCTO_APP"
  }
}
  • Mỗi {{tên_biến}} được thay bằng chuỗi JSON (string) khi gửi request
  • Giá trị inject luôn được escape đúng chuẩn JSON string
  • Nếu body/URL có {{...}}, dưới ô nhập hiện Resolved preview

Bước 3 — Gọi API và so sánh

A. Call Both & Compare

  • Gọi 1 lần Endpoint 1 và Endpoint 2 với biến hiện tại
  • Nếu có biến array, tool tự loop theo ưu tiên: jsArray → pcbRaw → biến trong body

B. Call Compare with Variable (khuyến nghị batch)

Mỗi hàng mapping gồm Array + Body field. Bấm + Thêm cặp array → body field để thêm hàng (không giới hạn cứng 2 hàng). Hàng đầu tiên quyết định vòng lặp.

HàngBắt buộcVai trò
Hàng 1 — Array (vòng lặp) + Body fieldCóBiến array quyết định số lần gọi API; inject array[i] vào body field
Hàng 2, 3, …Tùy chọnInject array khác cùng index i nếu còn phần tử

Cách chọn nhanh:

  1. Thêm ít nhất 1 biến array trong Variables (value bắt đầu bằng [)
  2. Tool tự gợi ý Array 1 = array đầu tiên, Array 2 = array thứ hai (nếu có)
  3. Body field gợi ý: pcbRaws → pcbRaw, icores → icore (bỏ s cuối)
  4. Kiểm tra / chỉnh dropdown nếu tên biến khác convention
  5. Bấm Call Compare with Variable

Logic vòng lặp:

  • Số lần gọi = độ dài Array 1 (không phụ thuộc Array 2)
  • Mỗi index i: luôn inject array1[i] → Body field 1
  • Nếu có Array 2 và i < độ_dài_array2 → inject array2[i] → Body field 2
  • Nếu i vượt độ dài Array 2 → bỏ qua inject array 2 (không báo lỗi)
  • Chỉ batch cả 2 API thành công mới đưa vào JSON gộp

Ví dụ: pcbRaws (213), icores (124), Array 1 = pcbRaws:

Index iInject
0 … 123pcbRaws[i] → pcbRaw và icores[i] → icoreRaw
124 … 212Chỉ pcbRaws[i] → pcbRaw

Ngược lại, Array 1 = icores (124) → chạy 124 lần, dùng pcbRaws[0..123] khi Array 2 = pcbRaws.

Chỉ 1 array: giữ 1 hàng mapping. Xoá hàng thừa bằng nút thùng rác. Sau khi chạy xong, tool tự compare.

Mẹo & lỗi thường gặp

Không thấy biến trong dropdown Array

Value phải là JSON array hợp lệ, bắt đầu bằng [.

Call Compare with Variable bị disable

  • Chọn đủ Array 1 và Body field 1
  • Array 1 phải là biến array hợp lệ trong Variables
  • Nếu chọn Array 2 thì phải chọn thêm Body field 2
  • Body Endpoint 1 nên có {{tên_field}} tương ứng

Hai array khác độ dài

  • Không lỗi — vòng lặp luôn theo Array 1
  • Array 2 chỉ inject khi index i còn trong phạm vi

Response rỗng

Kiểm tra preview, Failed batches, API trả JSON hợp lệ.

JSON quá lớn

Lite mode > 400 dòng; diff có phân trang.
